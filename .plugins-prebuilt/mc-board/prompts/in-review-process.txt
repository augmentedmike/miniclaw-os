## Git / Git Butler

The CLI is at: `/Applications/GitButler.app/Contents/MacOS/gitbutler-tauri` (aliased as `but` if set up).

Key commands (run with `-C <work_dir>` to target the right repo):
```
but -C <work_dir> status              # workspace overview — branches, uncommitted changes
but -C <work_dir> diff                # uncommitted changes across all virtual branches
but -C <work_dir> branch list         # all branches (applied and unapplied)
but -C <work_dir> show <branch>       # commits on a branch
but -C <work_dir> push <branch>       # push a branch to remote

```

If `but` isn't set up for the repo (no GitButler project found), fall back to standard git:
```

git -C <work_dir> status
```

## Review process

1. **Check git state** — use `but status` first, fall back to `git status`. What files changed? Are changes committed or staged? Reference specific file paths.

2. **Check the live site** — if card has a `verify_url`, hit it:
   ```
   curl -sI <verify_url>                    # HTTP status
   curl -s <verify_url> | grep "zinc\|rounded\|tracking"   # spot-check CSS classes in HTML
   ```

3. **Evaluate each acceptance criterion** — go through every `- [ ]` / `- [x]` item. For each:
   - Find the specific code or live output that satisfies it
   - Mark `- [x]` if confirmed with evidence, leave `- [ ]` if not

4. **Write review_notes** — be specific: file paths, line numbers, curl output, what passed, what failed.

5. **Decide: PASS or REJECT**

   **PASS** — all criteria met. Before moving to shipped, run the git ship sequence:

   ```
   # 1. Commit any uncommitted work on the current branch
   git -C <work_dir> add -A
   git -C <work_dir> status   # confirm what's staged
   git -C <work_dir> commit -m "feat(<plugin>): <summary of work done>" || echo "nothing to commit"

   # 2. Get the current branch name (may be a card ID branch from Git Butler)
   BRANCH=$(git -C <work_dir> rev-parse --abbrev-ref HEAD)

   # 3. Rebase onto latest main to pick up any concurrent changes
   git -C <work_dir> fetch origin
   git -C <work_dir> rebase origin/main
   # If rebase fails → REJECT the card with the conflict details

   # 4. If on a feature branch, merge into main then push
   if [ "$BRANCH" != "main" ]; then
     git -C <work_dir> checkout main
     git -C <work_dir> merge "$BRANCH" --ff-only
   fi
   git -C <work_dir> push origin main
   # If push fails → REJECT the card with the error
   ```

   Only after the push succeeds, move the card:
   ```
   openclaw mc-board update {{CARD_ID}} --review "<detailed review notes including commit hash>" --criteria "<criteria with all [x]>"
   openclaw mc-board update {{CARD_ID}} --log "Review PASS: <1-line summary>" --worker board-worker-review
   openclaw mc-board move {{CARD_ID}} shipped
   ```

   After shipping, check if the card's project has a `build_command` set and run it:
   ```
   # Get the project's build_command
   PROJECT_ID=$(openclaw mc-board show {{CARD_ID}} 2>/dev/null | grep "^project_id:" | awk '{print $2}')
   BUILD_CMD=$(openclaw mc-board project show "$PROJECT_ID" 2>/dev/null | grep "^\*\*Build command:\*\*" | sed 's/^\*\*Build command:\*\* `//;s/`$//')

   if [ -n "$BUILD_CMD" ] && [ -n "$PROJECT_ID" ]; then
     echo "Running build command for project $PROJECT_ID: $BUILD_CMD"
     LOG_FILE=~/.openclaw/logs/cards/{{CARD_ID}}.log
     echo "$(date -u +%Y-%m-%dT%H:%M:%SZ) [build] Running build_command: $BUILD_CMD" >> "$LOG_FILE"
     if eval "$BUILD_CMD" >> "$LOG_FILE" 2>&1; then
       echo "$(date -u +%Y-%m-%dT%H:%M:%SZ) [build] Build succeeded" >> "$LOG_FILE"
       openclaw mc-board update {{CARD_ID}} --log "Build succeeded: $BUILD_CMD" --worker board-worker-review
     else
       echo "$(date -u +%Y-%m-%dT%H:%M:%SZ) [build] Build FAILED — rejecting card back to in-progress" >> "$LOG_FILE"
       openclaw mc-board update {{CARD_ID}} --review "Build FAILED after ship: $BUILD_CMD — see ~/.openclaw/logs/cards/{{CARD_ID}}.log for details" --worker board-worker-review
       openclaw mc-board move {{CARD_ID}} in-progress --force
     fi
   fi
   ```

   **REJECT** — criteria not met, work broken, or git ship failed:
   ```
   openclaw mc-board update {{CARD_ID}} --review "REJECTED: <reason>" --criteria "<criteria with failed items as [ ]>"
   openclaw mc-board update {{CARD_ID}} --log "Review REJECT: <reason>" --worker board-worker-review
   openclaw mc-board move {{CARD_ID}} in-progress --force
   ```

6. **Flag non-blocking issues** in review_notes even on PASS (a11y gaps, follow-up items, TODOs left in code).
