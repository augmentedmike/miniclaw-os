## BLOCKED Protocol — When You Cannot Proceed

If you hit a genuine blocker (CAPTCHA visible, SMS verification required, credential missing, human decision needed):

**DO THIS — in order:**
1. Write a detailed memo explaining exactly what the block is:
   ```
   echo "$(date -u +%Y-%m-%dT%H:%M:%SZ) BLOCKED: <exact reason — e.g. 'Instagram shows CAPTCHA at signup step 3'>" >> ~/.openclaw/miniclaw/USER/memos/{{CARD_ID}}.md
   ```
2. Update the card notes with what was attempted and what's needed:
   ```
   openclaw mc-board update {{CARD_ID}} --notes "BLOCKED: <reason>. Mike must: <specific action>."
   ```
3. Add the hold tag so this card is NOT picked up again until the block is resolved:
   ```
   openclaw mc-board update {{CARD_ID}} --add-tags hold
   ```
4. Release:
   ```
   openclaw mc-board release {{CARD_ID}} --worker board-worker-in-progress
   ```

**NEVER release without hold if you made no progress.** A card that releases without hold will be picked up again in minutes, wasting API budget and causing visible side effects (browser flashing, etc.).

The only exception: if you made partial progress and can continue next session (e.g. wrote code but didn't test), release WITHOUT hold — the next agent will pick up where you left off via memos.

## Work Type Protocols

### DESIGN — tags: `design`, `ui`, `animation`, `texture`, `mockup`

**Tools available:**
- `openclaw mc-designer` — generate images, palettes, textures, mockups via Gemini
- `WebSearch` — find visual references, Dribbble/Behance examples, color theory, font pairings
- `WebFetch` — fetch specific design docs, CSS spec pages, font specimen pages
- `Bash` — run dev server, copy assets, install font/CSS packages
- `Read/Write/Edit` — implement design tokens, CSS, component files

Design cards require visual artifacts from mc-designer BEFORE writing any code. Do not skip this.

**HARD RULE: A design criterion that mentions "mc-designer" or "attached to card" CANNOT be checked `- [x]` unless:**
1. The PNG file physically exists at `~/.openclaw/miniclaw/USER/docs/media/designer/`
2. The attachment curl POST to `/api/cards/{{CARD_ID}}/attachments` returned HTTP 200
3. You ran `ls ~/.openclaw/miniclaw/USER/docs/media/designer/ | grep {{CARD_ID}}` and saw the file

Do NOT check the criterion based on code alone. Verify the file exists first.

**Step 1 — Generate reference assets first:**
```
# Color palette / mood board
openclaw mc-designer gen "Ghibli-inspired pastel color palette: sage greens, dusty pinks, warm creams, sky blues — watercolor swatch grid" -c {{CARD_ID}}-palette --role background
openclaw mc-designer composite -c {{CARD_ID}}-palette -o ~/.openclaw/miniclaw/USER/docs/media/designer/{{CARD_ID}}-palette.png

# Background texture or hero image
openclaw mc-designer gen "<description of background>" -c {{CARD_ID}}-bg --role background
openclaw mc-designer composite -c {{CARD_ID}}-bg -o ~/.openclaw/miniclaw/USER/docs/media/designer/{{CARD_ID}}-bg.png

# Component mockup (add elements on top of background)
openclaw mc-designer gen "<UI element description>" -c {{CARD_ID}}-mockup --role element -x 0 -y 0 --w 800 --h 600
openclaw mc-designer composite -c {{CARD_ID}}-mockup -o ~/.openclaw/miniclaw/USER/docs/media/designer/{{CARD_ID}}-mockup.png
```

**Step 2 — Attach every generated asset to the card:**
```
curl -s -X POST http://localhost:4220/api/cards/{{CARD_ID}}/attachments \
  -H "Content-Type: application/json" \
  -d '{"path":"~/.openclaw/miniclaw/USER/docs/media/designer/{{CARD_ID}}-palette.png","label":"Color palette reference"}'
```
Attach each PNG. The card must have at least one attached image before any code is written.

**Step 3 — Implement in code, referencing the generated assets:**
- Copy final assets to project `public/` directory
- Use exact color values extracted from the palette image
- Never hardcode hex values — always create CSS/Tailwind tokens first
- Implement components to match the mockup

**Step 4 — Verify visual output:**
- `npm run dev` must be running
- Open the relevant page and confirm visuals match the generated reference
- Screenshot or confirm in logs

Canvas naming: always prefix with card ID — `{{CARD_ID}}-palette`, `{{CARD_ID}}-bg`, `{{CARD_ID}}-mockup`.
Gemini API key is already loaded at openclaw startup — no setup needed.

mc-designer full command reference:
```
openclaw mc-designer gen "<prompt>" -c <canvas> --role background
openclaw mc-designer gen "<prompt>" -c <canvas> --role element -x <px> -y <px> --w <px> --h <px>
```
