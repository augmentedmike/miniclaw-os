== YOUR TASK ==

**Step 1 — Research**
Use bash tools to find relevant code, patterns, and context for this card. Look at:
- Related files in ~/.openclaw/miniclaw/plugins/ or ~/.openclaw/miniclaw/USER/projects/
- Existing patterns for similar work already done in the codebase
- Any referenced files, docs, or URLs in the card

**Step 2 — Fill out the card**
Run this command to save your research and notes:

  openclaw mc-board update {{CARD_ID}} \
    --research "<what you found: relevant files, patterns, dependencies, gotchas>" \
    --log "<1-2 sentence summary of what you found in this triage pass>"

If the card is missing an implementation plan, write one and add it:

  openclaw mc-board update {{CARD_ID}} \
    --plan "<step-by-step implementation plan based on your research>"

If the card is missing acceptance criteria, write them:

  openclaw mc-board update {{CARD_ID}} \
    --criteria "- [ ] <criterion 1>\n- [ ] <criterion 2>"

**Step 3 — Move to in-progress if ready**
A card is ready when it has ALL of:
- A clear problem_description
- A concrete implementation_plan
- Checkable acceptance_criteria (- [ ] items)

If ready:

  openclaw mc-board move {{CARD_ID}} in-progress

If NOT ready, note what is missing in --notes and leave it in backlog.

**Step 4 — Release**

  openclaw mc-board release {{CARD_ID}} --worker board-worker-web

Silent exit when finished. No summary messages.
